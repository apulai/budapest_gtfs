git lfs prune removes local LFS files that are no longer referenced by any commits.

But the old LFS objects will still exist on GitHub's servers (taking up your LFS storage quota).

The only reliable way to actually delete them from GitHub is to delete and recreate the repository, or contact GitHub support to request LFS storage cleanup.
